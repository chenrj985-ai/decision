# V7 GitHub重大消息面否决版

本版保留V7原有的机会筛选逻辑，没有启用V8的尾盘三次确认和大面积降级规则。

## 新增的安全层

- 自动扫描股票池近12天公告；
- 立案调查、财务造假、退市风险、预亏、业绩下修等直接一票否决；
- 业绩公告后下跌5%以上，按“市场确认明显不及预期”禁买；
- 减持、解禁、监管问询等事件风险达到45分，不得进入买入候选；
- 公告接口对某只股票读取失败时，只对该股票执行“消息面未知禁买”，不会影响其他股票；
- 消息面风险完整计入爆雷指数，技术高分、ETF强势、量价突然变好均不能抵消。

## 上传方法

将压缩包解压后的全部文件上传到GitHub仓库根目录，必须保留：

```text
.github/workflows/stock_auto.yml
```

在仓库设置中选择：

```text
Settings → Pages → Build and deployment → Source → GitHub Actions
```

然后进入Actions，手工运行一次 `V7 News-Safe Stock Auto Run`。

## 结论含义

- `消息面禁买`：明确重大利空或公告数据无法确认；
- `消息面回避`：解禁、减持、业绩预期差等风险较高；
- 这两类股票即使技术分很高，也不会成为买入候选。

本程序只能降低风险，不能保证完全避免下跌。正式操作仍建议只用小仓位。
