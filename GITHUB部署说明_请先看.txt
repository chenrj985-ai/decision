V6 GitHub全自动版部署说明
=========================

一、上传位置
必须把压缩包解压后的“内容”上传到GitHub仓库根目录。
根目录应直接看见：
main.py
github_publish.py
config.json
app
data
.github
index.html

不能出现：
仓库/StockDecisionSystem_V6_GitHub全自动版/main.py
这种多套一层目录的情况。

二、与之前GitHub保持一致的文件
1. mobile_latest.html
2. history/时间戳.html
3. latest_url.txt
4. latest_manifest.json
5. latest_decision.txt

latest_url.txt内容仍为JSON，例如：
{
  "latest_url": "https://用户名.github.io/stock/history/20260720_1440_00.html",
  "version": "20260720_1440_00",
  "beijing_time": "2026-07-20 14:40:00"
}

三、开启GitHub Pages
仓库 Settings
→ Pages
→ Build and deployment
→ Source选择 GitHub Actions

四、第一次运行
仓库 Actions
→ Stock Decision Auto
→ Run workflow
→ Run workflow

五、默认定时
北京时间工作日：
14:00
14:20
14:40
14:45
14:50
14:55
15:00

六、检查结果
最新网页：
https://用户名.github.io/仓库名/mobile_latest.html

最新历史地址：
https://用户名.github.io/仓库名/latest_url.txt

运行状态：
https://用户名.github.io/仓库名/status.json

七、程序失败时
GitHub不会删除之前成功的页面。
status.json会写入本次失败信息。
进入Actions打开失败任务，即可查看具体错误。

八、特别注意
GitHub Actions必须具备写入权限。
如果提交生成文件时报403：
Settings → Actions → General
→ Workflow permissions
→ 选择 Read and write permissions
→ Save
